# architectural_decision_logs.md
## Example Log
```md
### 016. Short Decision Title
- **Status:** Accepted | Rejected | Proposed | Accepted, current path | Accepted, not implemented
- **Date:** YYYY-MM-DD
- **Context:** Explain the problem, constraint, or situation that forced a decision.
- **Decision:** State the actual decision clearly. Include the chosen tool, pattern, architecture, or policy.
- **Consequences:**
  - **Positive:** What improved because of this decision.
  - **Positive:** Another benefit, if relevant.
  - **Negative:** Tradeoffs, costs, or risks.
- **Rejected Alternatives:**
  - **Alternative Name:** Why this was not chosen.
  - **Another Alternative:** Why this was not chosen.
```
